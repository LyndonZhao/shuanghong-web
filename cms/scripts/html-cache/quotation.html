③AI 报价系统

案例背景

本页是 广州双泓科技有限公司 在 海外窗户工程承接客户 上的真实落地案例。每家工厂的工艺、客户惯例、报价口径都不同，本案例供参考思路，实际落地需按 P0–P3 流程定制（详见 合作流程）。

一句话定位

给销售员一个 项目级综合 $/㎡ 推荐价（含加项），保证不亏本，对齐历史相似项目。把 2000 份散落的历史报价转化为系统可检索的知识资产。

系统永远不替代销售员判断，永远不自建工厂 BOM——工厂 ¥ 是黑盒输入，系统不复刻、不介入工厂内部成本核算。

端到端流程

[销售员]  上传 工厂报价xlsx + 报价说明docx (+ 可选 PDF)
   ↓
[系统]    解析输入 → 项目分档 → 查询历史相似项目
   ↓
[系统]    计算 不亏本红线 + 推荐价 + 议价底线 + 加项明细
   ↓
[系统]    生成"报价决策卡"（推荐价 + 红线 + 历史邻居 + 加项）
   ↓
[审批人]  按项目档位路由审批（飞书/企微通知）
   ↓
[系统]    审批通过 → 渲染 V1.xlsx（与现行模板一致）→ 销售员下载发客户
   ↓
[系统]    本次报价 + 议价 + 成交结果，自动沉淀进历史记忆库（下次更准）

核心能力

1. 报价文件解析

文件类型处理
工厂报价 xlsx（中文 ¥）表结构识别 + 行级抽取（系列 / 规格 / 数量 / 单价）
报价说明 docx系统、玻璃配置、变体、排除项识别
客户附件 PDF（工图，可选）项目元信息提取（地址、规模、备注）

2. 项目分档器

按总平米数把项目分到 5 个档位，不同档位独立学习，避免大 / 小项目策略互相污染：

档位范围典型场景
XS< 50 ㎡单户改造
S50–200 ㎡小型住宅
M200–800 ㎡标准住宅 / 联排
L800–3000 ㎡大型住宅 / 小型商业
XL> 3000 ㎡大型商业 / 项目集

档位边界可由项目经理在系统内调整。

3. 历史记忆库

维度设计
沉淀容量2000 份历史项目（一次性灌入）+ 每次新报价
检索方式语义检索（项目地点 / 客户类型 / 玻璃配置 / 五金品牌）+ 档位过滤
邻居数量每次返回 3–5 个最相似项目，附 $/㎡ 与成交结果
历史影响力邻居的 $/㎡ 作为推荐价 anchor，结合档位系数与不亏本红线综合得出

4. 推荐价引擎

推荐价 = max(
    不亏本红线,
    历史邻居 $/㎡ × 档位系数 × 客户类型系数 × 加项系数
)

每个推荐数字都能追到：

  • 工厂行（哪几行 BOM）
  • 档位（XS/S/M/L/XL）
  • 系数来源（客户类型、加项、市场调整）
  • 历史邻居（具体哪几个项目）

5. 不亏本约束器

项内容
公式红线 = 工厂总成本 × (1 + 最低毛利率)
最低毛利率按项目档位 / 客户类型差异化设定，由业务负责人维护
强约束任何报价（推荐价、议价底线）永不低于红线
异常告警销售在系统外私自报价低于红线 → 飞书告警

6. 加项综合器

加项类型处理
纱窗命名归一 + 标准计价
木窗套按平米 / 按樘归一
副框系列识别 + 计价
五金升级品牌等级映射 + 加价系数
特殊涂层 / finish颜色规范化 + finish 加价

输出 综合 $/㎡（含全部加项）+ 详细加项明细（合规呈现）。

7. 报价模板渲染

输出与现行 V1 / PI 模板 一致 的 Excel：

输出用途
V1.xlsx给客户的首版报价
决策卡（系统页面）内部销售 / 销售主管查看
PI.xlsx（议价后）议价后的正式报价单

模板适配：贵方现有 V1 / PI 模板可作为样板，1 周内完成适配。

8. 审批路由

触发默认审批链
XS / S 档 + 高于推荐价销售本人
M 档销售主管
L / XL 档销售主管 + 销售总监
议价至接近红线（< 红线 × 1.05）销售总监 + 财务
议价至红线销售总监 + CEO

9. 工厂报价版本管理

工厂涨价 / 改报价 → 销售端是否同步调价，自动监控：

项内容
多版本支持同项目可上传多版工厂报价（v1, v2, ...）
自动重算新工厂报价进来 → 自动重算红线 + 推荐价
利润影响差异输出"如果按 v2 工厂报价，利润影响 -8.3%"等量化报告

10. 滚动学习

维度设置
自动沉淀每次成交（V1 / PI / 实际成交价）自动入库
系数演化每周回学新数据，更新各档位的"客户类型系数 / 加项系数"
月度报告各档位的推荐价命中率、议价让步率、亏本风险率

关键承诺

维度承诺实现机制
不亏本任何报价永不低于工厂成本 × 最低毛利数学硬约束，永不绕过
不替代销售判断系统给推荐价，销售决定最终报价推荐价 + 议价底线 + 邻居参考，但最后一步由销售确认
不自建工厂 BOM永不复刻工厂内部成本核算工厂 ¥ 视为黑盒输入；系统不输出"工厂应该改 X"
可解释每个推荐数字能追到工厂行 / 档位 / 系数 / 邻居全程留痕，财务 / 审计可追溯
不被同行复用贵方历史项目、议价底线、客户类型系数不进入任何跨客户共享池物理隔离专属租户

不在本次范围

能力推迟原因启动条件
工图自动识别与图纸审核系统重叠，建议合并购买看 双系统联动
自建工厂 BOM / 物料库工厂内部黑盒，应由工厂负责永不
替代销售员议价决策议价是商务判断，不是数学问题永不
自动出 PI 合同正文合同条款属法务范畴，不在 MVP业务反馈后单独立项

实施计划（速查）

阶段工期主要工作验收标准
P0 样本治理2 周收 30–50 份典型历史项目 + 档位边界初始化历史邻居召回率达约定门槛
P1 解析与分档2 周文件解析 + 项目分档 + 历史记忆库上线上传 → 出邻居跑通
P2 推荐价与红线3 周推荐价引擎 + 不亏本约束 + 加项综合 + V1.xlsx 渲染销售员端到端跑通"上传 → 出 V1.xlsx"
P3 审批与版本1 周审批路由 + 工厂报价多版本管理至少 3 个 fixture 跑通
P4 滚动学习2 周系数自动演化 + 月度报告首份月度报告产出

总工期 8–10 周。
