②AI 图纸审核

案例背景

本页是 广州双泓科技有限公司 在 海外窗户工程承接客户 上的真实落地案例。每家工厂的工艺、规范、客户惯例都不同，本案例供参考思路，实际落地需按 P0–P3 流程定制（详见 合作流程）。

一句话定位

机器初审，工程师终审。把审图工程师从"逐张翻图核对尺寸/数量/玻璃/五金"中解放出来，只复核机器揪出的关键问题；客户惯例与错误模式沉淀为可复用资产。

系统永远不替代工程师签字，永远不"自作主张"——只问问题，不给建议值。

端到端流程

[销售员]  上传 图纸PDF + 门窗表xlsx + 规格说明docx (+ 可选拍照)
   ↓
[系统]    分类文件 → 解析 → 抽取每樘门窗的字段（带"来源 + 把握度"）
   ↓
[系统]    多视图对账（尺寸/数量是否一致）+ 必填清单 + 规范规则 + 客户规则
   ↓
[系统]    生成"审图卡片"（红黄蓝问题清单 + 证据截图）
   ↓
[系统]    把红黄问题翻译为人话问句，推到销售（飞书/企微）
   ↓
[销售员]  把问题转给客户 → 客户答复 → 销售在平台回填
   ↓
[系统]    自动重新对账 + 重新检查规则 → 红色问题逐条消化
   ↓
[工程师]  在平台复核剩余红色问题 → 签字
   ↓
[系统]    审核通过 → 把"结构化门窗清单"喂给报价系统
   ↓
[系统]    本次审图全过程沉淀进经验记忆库（错误模式、客户惯例、豁免理由）

核心能力

1. 多源文件解析

文件类型处理失败降级
图纸 PDF（矢量 / 扫描）按视图分页解析扫描页走 OCR
门窗表 xlsx / PDF表结构识别 + 单元格抽取单元格 OCR
规格说明 docx段落 + 客户默认值识别—
客户附件 PDF内容分类（合同 / 节点 / 五金品牌）—
手机拍照 / 手写视觉模型识别 + 低置信度标注标"待人工核"

关键设计：每个抽取字段都带 来源 + 置信度。低置信度字段不直接进主结果，必须由销售 / 工程师人工确认。绝不在低质输入上猜值。

2. 多视图对账

字段类型容差不一致时的级别
宽 / 高（毫米）±10mm 或 ±0.5%（取大）🔴 红色（数字直接影响成本/制造）
数量必须相等🔴 红色
系列 / 型材体系同义词归一后必须相同🔴 红色
玻璃 / 颜色 / finish同义词归一后必须相同🟡 黄色
开启方向字符串相同🟡 黄色
安装节点引用引用 ID 必须存在于附图🟡 黄色
命名风格仅记录🔵 蓝色

每条对账问题必须列出 所有来源的取值 + 证据指针（哪个 cell / 哪页哪个区域）+ 裁剪图证据。

3. 三层规则库

层内容维护方例子
L1 全局必填任何门窗必须有的字段系统内置系列 / 宽 / 高 / 数量 / 玻璃 / 开启
L2 国别规范国家 / 体系特有要求系统内置（按 region）AS 2047 防风、GB 50189 热工、EN 14351 CE
L3 客户专属客户合同 / 惯例项目经理在系统内维护"客户 X 所有玻璃必须 LowE+Argon"

每条规则可见可审：触发条件、严重度、违反提示、依据来源。客户 / 项目级规则一次录入，永久套用。

4. 问题分级与硬闸门

级别含义默认行为谁能解除
🔴 红色（关键）数字 / 数量 / 系列错配，或安全规范不满足强制阻断，未消化禁止进入报价 / 生产工程师签字
🟡 黄色（重要）玻璃 / 颜色 / finish 缺失或不一致默认阻断，可豁免（必须写理由）销售主管或工程师
🔵 蓝色（提示）命名风格、可推断的非关键字段仅记录，不打扰—

强约束：系统永不允许带未解决红色问题的项目流向报价 / 生产。

5. 自动澄清问句生成

把技术问题翻译为销售看得懂的人话问句，推到协作平台：

📩 项目 P-2026-0123 有 7 个问题需要找客户确认：

  1. W-103 的高度：门窗表写 1500，立面图量出 1480，以哪个为准？
  2. L2 主卧 4 樘窗的玻璃配置 未标注，请确认是否为 LowE+Argon？
  3. D-201 防火门的五金品牌 不在 NCC 认证清单内，请客户提供认证文件或更换品牌。 ...
项内容
翻译策略不出现技术术语；附原图截图作为客户参考
推送渠道飞书 / 企业微信 / 钉钉 / 平台站内消息
收答机制销售在平台回填客户答复 → 系统自动重新对账 → 问题消化
强约束系统 只问问题，不给建议值——避免销售习惯性点同意导致错误传导

6. 工程师终审与审批路由

按问题严重度 × 客户级别默认路由（可在系统中调整）：

触发默认审批链
全蓝色销售本人确认
≤ 3 黄色销售主管单签
含红色 / >3 黄色销售主管 + 审图工程师双签
含红色 + 涉及防火 / 防风 / 热工审图工程师 + 技术经理双签（工程师必须线下复核图纸）
VIP / 战略客户默认链 + 销售总监知会

7. 经验记忆库与滚动学习

维度设置
自动沉淀客户惯例（如"客户 X 默认 LowE+Argon"）、错误模式、豁免理由
自动套用同客户 / 同区域新项目自动注入历史惯例，无须重复录入
月度报告各规则触发率、误报率、豁免率 → 业务团队评估规则是否需要收紧 / 放宽
异常告警某规则误报率超出约定阈值持续一段时间 → 飞书告警人工评审

与现状 / 通用方案的对比

维度贵方现状通用 PDF 阅读器 / 简单 OCR商业 CAD 审图软件本系统
入门成本资深工程师带教工具需自己写脚本强制改贵方图纸格式销售 / 工程师短期培训即可上手
单项目审图耗时工程师人工逐张部分自动，仍要人工拼受限于软件场景机器初审 + 工程师只复核红色问题
多视图对账人眼不支持部分支持（依赖统一格式）自动对账 + 证据截图
必填字段凭工程师经验无强制模板三层规则库（全局+国别+客户）
客户惯例沉淀凭记忆无无自动学习 + 自动套用
规范条款散落工程师脑中无部分内置AS / GB / EN 模块化加载 + 可定制
客户图纸适配工程师手工适配严格依赖图纸格式客户必须改格式多源解析 + OCR / 视觉 + 三级降级
报价系统衔接人工传无部分有 ERP 接口状态契约 + 自动交付 + 审核未过强制拒收
数据归属贵方自有通常本地部分云端物理隔离专属租户、绝不外流
替代工程师签字——部分软件试图替代（风险）永不替代，定位是"机器初审、工程师终审"

关键承诺

维度承诺实现机制
机器初审，工程师终审AI 永远不替代签字红色问题必须工程师人工确认；签字记录留底
AI 不自作主张系统 只问问题，不给建议值澄清问句模板严格约束；不替客户填字段；不替工程师改图
不替代专业判断结构 / 防水 / 热工等专业判断由工程师承担系统不输出此类结论；规则库不涵盖专业计算
可解释每条问题附"证据截图 + 依据"来源标签 + 裁剪图 + 规则编号全程可追
可干预工程师可手动新增问题、可豁免黄色（写理由）系统记录所有人工干预，作为规则演化依据
不丢数据全部图纸 / 审图记录物理隔离在贵方专属租户平台级多租户隔离
不被同行复用贵方审图规则、客户惯例、错误模式不进入任何跨客户共享池不做行业基准、不做客户间数据互通
