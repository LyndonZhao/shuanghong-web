## 1. 开始之前

### 1.1 这个后台是干什么的?

这是**双泓科技官网**的内容管理后台。运营人员在这里修改的所有内容,都会**实时反映**到对外的官网上(<https://www.shuanghongtech.com> 或当前 Next.js 前端服务地址)。

**它能干什么:**

| 你想做 | 在后台做什么 |
|---|---|
| 改首页的大标题、按钮 | 内容管理器 → 首页 |
| 上线一个新的解决方案 | 内容管理器 → 解决方案 → 创建新条目 |
| 看客户提交的询盘 | 内容管理器 → 询盘 |
| 换公司 Logo、改导航菜单 | 内容管理器 → 站点设置 |
| 换 Banner/产品图 | 媒体库 |
| 改 SEO 关键词 | 每个页面下方的"SEO"区块 |
| 给新同事开账号 | 找超级管理员,见 [§ 17 应急与联系](#17-应急与联系) |

**它不能干什么:**

- 不能改前端代码(页面布局、配色、字体)— 这些需要找开发同事。
- 不能直接看网站访问量 — 请去百度统计或 Google Analytics 后台。
- 不能"撤销"已经保存的修改 — Strapi Community 版没有版本历史(这是付费功能)。**所以重要修改前一定要拷贝一份原文本备份。**

### 1.2 你需要的账号

- **邮箱**(由超级管理员分配,例如 `your-name@shuanghongtech.com`)
- **密码**(初次登录后强烈建议立即修改)

如果还没账号,联系超级管理员 `hui85201@gmail.com`。

### 1.3 浏览器要求

推荐 **Chrome / Edge / Firefox 最新版**。不支持 IE。建议使用 1920×1080 或更高分辨率,小屏会被压缩。

## 3. 核心概念:三类内容

在动手编辑前,先理解 Strapi 的三个核心概念,**理解了它们后面所有操作都会变得简单**。

### 3.1 单类型(Single Type)

**只有一份的内容**,就像"首页"在网站上永远只有一个。本项目的单类型有:

| 单类型 | 对应网站 URL |
|---|---|
| **首页** | `/`(网站首页) |
| **多模态交互** | `/multimodal` |
| **AI 应用** | `/ai-application` |
| **关于我们** | `/about` |

**操作方式**:点进去 → **直接编辑** → 点击右上角「保存」。不需要"新建"。

### 3.2 集合类型(Collection Type)

**可以有很多条**的内容,像一个表格,每条记录是一行。本项目的集合类型有:

| 集合类型 | 实际作用 | 现有条数 |
|---|---|---|
| **解决方案** | 9 个产品方案卡片,首页和栏目页会列出 | 9 |
| **询盘** | 客户通过网站表单提交的咨询线索 | 持续增长 |
| **站点设置** | ⚠️ 虽然是"集合"类型,但**只允许有 1 条**(全站配置) | 1 |

**操作方式**:点进去 → 看到列表 → 「创建新条目」/ 「编辑」/ 「删除」。

### 3.3 组件(Component)

**可复用的字段块**,被嵌入到内容类型中。本项目有 2 个组件:

| 组件 | 用在哪里 | 字段 |
|---|---|---|
| **SEO 元数据** | 首页、多模态交互、AI 应用、关于我们、解决方案 | 标题/描述/关键词/分享图 |
| **联系方式** | 关于我们 | 电话/邮箱/地址/微信号 |

你看到表单里有一块"卡片",标题是「SEO」或「Contact」,点击展开就能编辑里面的字段。

## 5. 修改"首页"内容

首页是访客看到的**第一屏**,所有字段都直接影响第一印象。

### 5.1 进入首页编辑

**内容管理器** → 左侧栏「单类型」分组 → **首页**。

### 5.2 字段一览表

| 字段名(英文) | 中文含义 | 类型 | 必填 | 现在默认值 |
|---|---|---|---|---|
| `heroTitleLine1` | Banner 主标题第 1 行 | 短文本 | ✅ | "汽车与机器人领域的多模态交互专家" |
| `heroTitleLine2` | Banner 主标题第 2 行 | 短文本 | ✅ | "制造及建筑业的智能AI应用普及者" |
| `heroSubtitle` | Banner 副标题(支持加粗、链接) | 富文本 | ❌ | (空) |
| `ctaPrimaryLabel` | 主按钮文字 | 短文本 | ❌ | "多模态交互方案" |
| `ctaPrimaryLink` | 主按钮链接 | 短文本 | ❌ | "/multimodal" |
| `ctaSecondaryLabel` | 副按钮文字 | 短文本 | ❌ | "智能AI应用解决方案" |
| `ctaSecondaryLink` | 副按钮链接 | 短文本 | ❌ | "/ai-application" |
| `heroBackground` | Banner 背景图(单张) | 媒体 | ❌ | (待上传) |
| `seo` | SEO 元数据组件 | 组件 | ❌ | (空) |

### 5.3 操作步骤

#### 步骤 1:改文字

直接在文本框里改即可。**两行主标题建议每行 12~16 字**,太长会自动换行,影响美观。

#### 步骤 2:换 Banner 图

1. 点击「heroBackground」字段下的「上传媒体文件」按钮。
2. 弹出窗口:
   - **从计算机选**:拖图片进去或点「浏览」。
   - **从媒体库选**:已上传过的图片直接选。
3. 选好后点「完成」。
4. ⚠️ **推荐尺寸**:1920×800 像素,JPG 格式,**单图小于 1MB**(影响打开速度)。

#### 步骤 3:改 CTA 按钮

- **Label = 按钮文字**(如"立即咨询")。
- **Link = 跳转地址**。两种写法:
  - 内部页:`/multimodal`、`/about`(以 `/` 开头)。
  - 外部页:`https://其他网址`(完整 URL)。

#### 步骤 4:保存

右上角「保存」按钮。**首页没有"发布"环节,保存即生效**(本项目所有页面都关闭了草稿/发布机制,见 [§ 15 发布与生效机制](#15-发布与生效机制))。

### 5.4 改完后怎么验证?

在浏览器**新开一个窗口**,打开网站首页(<http://localhost:3000> 或正式域名),按 **Ctrl + F5** 强制刷新缓存,确认改动已生效。

## 6. 修改"多模态交互"页

对应网站的 `/multimodal` 栏目页。

### 6.1 字段一览

| 字段名 | 中文含义 | 类型 | 默认值 |
|---|---|---|---|
| `coreAdvantageTitle` | 核心优势标题 | 短文本 | "核心优势:端云协同的多模态交互" |
| `coreAdvantageDesc` | 核心优势描述 | 富文本 | (空) |
| `solutions` | 关联的解决方案(可多选) | 关联 | (默认全部 multimodal 类) |
| `seo` | SEO 元数据 | 组件 | (空) |

### 6.2 重点:如何关联解决方案

`solutions` 字段是**关联字段**,负责告诉前端"这个页面要展示哪些方案卡片"。

操作:

1. 在「solutions」字段处,点「添加关联条目」。
2. 弹出**解决方案列表**,**勾选**要展示的几张方案卡(可多选)。
3. 拖拽可调整**显示顺序**。
4. 点「完成」→ 「保存」。

> 💡 **运营惯例**:多模态交互页通常只挂 `category = multimodal` 的方案;AI 应用页挂 `category = ai_application`。如果挂错了,前端筛选仍会过滤一遍,但建议手动归类清晰。

### 6.3 富文本编辑器小提示

`coreAdvantageDesc` 支持:

- **加粗 / 斜体**(顶部工具栏或 Ctrl+B / Ctrl+I)
- **标题层级 H1~H6**(下拉选)
- **列表**(无序、有序)
- **链接**(选中文字 → 点链接图标 → 粘 URL)
- **图片**(嵌入正文,会上传到媒体库)
- **代码块**(技术文档用)

⚠️ **不要从 Word 直接复制粘贴**,会带入大量乱样式。建议先粘到记事本→再粘到富文本框。

## 9. 维护"解决方案"列表

解决方案是**整站的核心内容资产**,影响:

- 首页方案展示区
- 多模态交互页方案区
- AI 应用页方案区
- 询盘表单的 `interest` 字段(`multimodal` / `ai` / `other`)

### 9.1 当前已有 9 条(2026-06-13)

| ID | 分类 | 顺序 | 标题 |
|---|---|---|---|
| 1 | multimodal | 1 | 导览产品 |
| 2 | multimodal | 2 | 文旅智能互动 |
| 3 | multimodal | 3 | 企业服务 |
| 4 | multimodal | 4 | 汽车座舱 |
| 5 | ai_application | 5 | AI 辅助专利撰写 |
| 6 | ai_application | 6 | AI 辅助报价系统(门窗柜子装修) |
| 7 | ai_application | 7 | AI 辅助图纸审核系统(装修) |
| 8 | ai_application | 8 | 数字营销系统 |
| 9 | ai_application | 9 | 整车 AUTOSAR CP 研发自动化 |

### 9.2 字段一览

| 字段 | 含义 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| `title` | 方案标题 | 短文本 | ✅ | **整库唯一**,不可重复 |
| `category` | 分类 | 枚举 | ✅ | 只能选 `multimodal` 或 `ai_application` |
| `description` | 方案简介 | 富文本 | ❌ | 卡片正文 |
| `coverImage` | 封面图 | 媒体(单张) | ❌ | 推荐 800×500,JPG/PNG/WebP |
| `slug` | URL 别名 | UID | ❌ | 自动从 `title` 生成,如 `tour-guide` |
| `order` | 显示顺序 | 整数 | ❌ | **数字越小越靠前**,默认 0 |
| `tags` | 标签(JSON) | JSON | ❌ | 例:`["机器人","展馆"]` |
| `seo` | SEO 元数据 | 组件 | ❌ | 方案详情页用 |

### 9.3 新增一个解决方案

1. 内容管理器 → **解决方案** → 右上角「**创建新条目**」。
2. 必填:
   - **title**:产品/方案名(中文,例如"AI 客服机器人")。
   - **category**:在下拉选 `multimodal` 或 `ai_application`。
3. 强烈建议填:
   - **description**:200~400 字简介。
   - **coverImage**:上传封面图(没图就用默认占位图)。
   - **order**:看现有列表最大 order 是 9,新方案就填 10。
4. **slug** 字段会**自动从 title 生成**(中文会被转拼音或英文)。如果想自定义,点旁边的「编辑」即可。
5. **tags**(可选):JSON 数组写法,例:

   ```json
   ["机器人", "展馆", "智能讲解"]
   ```

6. 右上角「保存」→ 立即生效。

### 9.4 调整顺序

⚠️ Strapi 没有"拖拽排序"的内置功能。**调整顺序 = 改 order 数字**:

- 想让某条置顶 → 把它的 order 改成 0。
- 想让多个连续排序 → 1,2,3,4...
- 数字相同 → 按创建时间排序(不可控)。

> 💡 **建议**:每次新增方案,order 按 10、20、30 这样**留间隔**,后续插队就改成 15、25 等,不用全表重排。

### 9.5 下线一个解决方案

两种选择:

| 操作 | 后果 | 适合场景 |
|---|---|---|
| **删除** | 数据永久消失,前端立即看不到 | 完全报废、写错了 |
| **改 category 到一个不展示的值** | ⚠️ 当前不支持,只有 2 个枚举 | — |
| **设 order 为很大的数(如 9999)** | 排到最后,但仍然会显示 | — |

**推荐做法**:暂时不展示 → **删除**(因为后端已经做了关联清理)。
长期想保留草稿 → 当前项目的解决方案 schema **关闭了 draftAndPublish**,所以没有"草稿"概念。如果有此需求请联系开发。

### 9.6 修改现有方案

## 10. 处理"询盘"客户表单

询盘是**客户通过官网表单主动提交**的咨询线索。

### 10.1 谁会收到通知?

每次有新询盘提交,系统会**自动发邮件**到:

- **收件人**:环境变量 `SMTP_TO`(默认 `sales@shuanghongtech.com`)
- **抄送**:无
- **发件人**:`noreply@shuanghongtech.com`(走腾讯企业邮箱 `smtp.exmail.qq.com`)
- **邮件内容**:客户姓名、公司、邮箱、感兴趣方向、留言、来源页 + 一键跳后台链接

> 💡 如果没收到通知邮件:
> 1. 检查垃圾邮件夹。
> 2. 在后台手动查最新询盘(本节继续)。
> 3. 联系开发同事确认 `SMTP_*` 环境变量配置正确。

### 10.2 字段一览

| 字段 | 中文 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| `name` | 客户姓名 | 短文本 ≤50 | ✅ | 客户填的 |
| `company` | 公司 | 短文本 ≤100 | ✅ | 客户填的 |
| `email` | 邮箱 | 邮箱 | ✅ | 客户填的(主要联系方式) |
| `phone` | 电话 | 短文本 | ❌ | 客户填的 |
| `interest` | 感兴趣方向 | 枚举 | ❌ | `multimodal` / `ai` / `other` |
| `message` | 客户留言 | 长文本 ≤1000 | ✅ | 客户填的 |
| `sourcePage` | 来源页 URL | 短文本 | ❌ | 系统自动记录 |
| `status` | **处理状态** | 枚举 | ✅ | **运营人员维护** |
| `handledBy` | 负责人 | 关联管理员 | ❌ | **运营人员选自己** |
| `handledAt` | 处理时间 | 日期时间 | ❌ | **联系客户后填** |
| `notes` | 内部备注 | 长文本 | ❌ | **跟进记录,客户看不到** |

### 10.3 处理状态(status)的四个值

| 值 | 中文 | 含义 | 何时改 |
|---|---|---|---|
| `pending` | 待处理 | **默认值**,新询盘 | 客户刚提交 |
| `processing` | 处理中 | 销售已认领正在评估 | 自己接手后立刻改 |
| `contacted` | 已联系 | 已电话/邮件联系到客户 | 实际联系上后 |
| `closed` | 已结案 | 不再跟进(成单/客户拒绝/无效) | 流程终结 |

### 10.4 标准跟进流程(运营 SOP)

```
邮件通知 → 进后台 → 看详情 → 评估
                              ├─ 真实客户 → 改 status=processing,handledBy=自己 → 保存
                              │                ↓
                              │           联系客户(电话/邮件)
                              │                ↓
                              │           改 status=contacted,filled handledAt=今天,notes=跟进结果
                              │                ↓
                              │           成单 / 失败 → 改 status=closed,补充 notes
                              │
                              └─ 垃圾询盘 → 直接改 status=closed,notes="spam"
```

### 10.5 重要约定

- ⚠️ **不要删除询盘记录**!即使是垃圾询盘,也只 `status=closed`。
  原因:法务追溯、统计转化率、防止误删真实客户。
- ⚠️ `notes` 字段是**内部使用**,**绝不能写客户隐私吐槽**(数据库泄漏风险)。
- ✅ 每天**至少查 1 次**待处理询盘(`status=pending`),建议早上 9 点。

### 10.6 询盘列表过滤技巧

在列表页右上角:

- **筛选**:点「筛选条件」→ 选 `status = pending` → 只看新询盘。
- **排序**:列表表头点击日期列,可按时间倒序看最新的。
- **搜索**:右上角搜索框,可按客户姓名、公司、邮箱关键字搜。

### 12.3 文件命名规范

- ✅ **推荐**:小写英文 + 短横线,如 `hero-banner-2026q3.jpg`
- ❌ **避免**:中文名(部分浏览器会编码异常)、空格、特殊符号
- 上传前**改文件名**,Strapi 不会自动改

### 12.4 图片优化建议

| 用途 | 推荐格式 | 推荐尺寸 | 文件大小 |

| Logo | SVG | — | ≤ 50 KB |
| SEO 分享图(og:image) | JPG | 1200×630 | ≤ 500 KB |
| 富文本插图 | JPG / PNG | 宽度 ≤ 1200 | ≤ 500 KB |

> 💡 用 [TinyPNG](https://tinypng.com) 或 [Squoosh](https://squoosh.app) 上传前压一下,可瘦 50%~70%。

### 12.5 文件夹管理

  /solutions     ← 方案封面

**好处**:所有引用了这张图的页面会**自动用新图**,不用一个个改引用。

## 15. 发布与生效机制

⚠️ **本项目的关键约定**:

| 内容类型 | 草稿 / 发布机制 | 保存即生效? |
|---|---|---|
| 首页 / 多模态 / AI / 关于我们 / 解决方案 / 站点设置 / 询盘 | **关闭**(`draftAndPublish: false`) | ✅ 是 |

**意思是:**

- ✅ **点「保存」=「发布」**,改动立刻同步到前端站点。
- ❌ **没有"草稿"状态**,改一半的内容也会立刻上线。
- ❌ **没有"取消发布"**,只能再次编辑或删除。

**因此运营务必:**

1. **重大文案修改前,先备份原文**(自己存到本地 Word 或备忘录)。
2. **批量改动时,先一气改完再保存**,而不是改一半挂着。
3. **重要发布前**(尤其是首页),**先打开网站确认旧版本截图**,以便回滚参考。

> 💡 如果未来确实需要"草稿/审批"工作流(例如新闻动态),可让开发同事**单独**为该内容类型开启 `draftAndPublish: true`,不影响现有页面。

## 16. 常见问题 FAQ

### Q1:为什么我改了首页内容,网站上没变化?

按以下顺序排查:

1. **浏览器缓存**:**Ctrl + Shift + R**(强制刷新)。
2. **CDN 缓存**:如果用了 CDN(生产环境),需要在 CDN 控制台「刷新缓存」。
3. **前端 SSR 缓存**:Next.js 的 ISR 默认 60 秒重新生成,等 1 分钟再看。
4. **是否点了「保存」**:右上角按钮变灰色 = 已保存。
5. **是否登错了环境**:开发后台 vs 生产后台是两套数据,确认 URL。

### Q2:我点「保存」时报错"validation error"

最常见 3 个原因:

- **必填字段为空**:红色字段名带 \* 号,补全即可。
- **唯一字段重复**:如 `solution.title` 已存在同名,改个名。
- **JSON 字段格式错**:`navMenu` 或 `tags` 写法不合法(忘逗号、用了中文标点),用 <https://jsonlint.com> 校验。

### Q3:不小心删了一条解决方案怎么办?

- **本地开发环境**:SQLite 数据库文件可能有备份(联系开发)。
- **生产环境**:依赖运维的**每日数据库备份**恢复(联系运维)。
- **建议**:**今后删除前,先截图保存字段内容**,真要删确认无引用再删。

### Q4:为什么富文本编辑器粘进去的内容样式很丑?

很可能从 Word / WPS / 邮件复制带了样式。处理方法:

1. 富文本框右上角点「**清除格式**」按钮。
2. 或者:先粘到**记事本**(去掉所有样式)→ 再粘到编辑器。
3. 或者:**直接在编辑器里写**,用工具栏加粗/标题。

### Q5:图片上传超大失败怎么办?

- Strapi 默认上传上限是 200MB,但**建议单图 < 1MB**。
- 太大请**先用 TinyPNG 压缩**。
- 如果是必须的高清大图(如产品高清图),联系开发调大上限,但**会拖慢页面**。

### Q6:可以批量导入解决方案吗?

- 后台无 UI 批量导入功能。
- 可以让开发同事用**项目内的种子脚本**改造批量插入。
- 单次几十条以下,**逐条手动创建更安全**(避免错乱)。

### Q7:网站显示的中文乱码 / 显示英文了

- 后台界面英文:右上角头像 → 资料设置 → 「界面语言」改成「简体中文」。
- 前端网站乱码:检查浏览器编码是否 UTF-8;不是,联系开发。
- 前端显示英文 "Title goes here":这是**字段还没填默认值**,在内容管理器里补充即可。

### Q8:邮件通知收不到询盘提醒

可能原因:

1. **垃圾邮件夹** — 把 `noreply@shuanghongtech.com` 加白名单。
2. **SMTP 配置错** — 联系开发检查 `.env` 的 `SMTP_HOST / SMTP_USERNAME / SMTP_PASSWORD`。
3. **SMTP_TO 收件箱满了** — 清理 `sales@` 邮箱。
4. **客户提交失败** — 看后台**询盘列表**确实没有新数据 → 前端表单可能挂了。

### Q9:我能让客户看到我们的内部备注 `notes` 吗?

**不能**。`notes` 字段**没有暴露给 Public API**,只有后台管理员可见。
公开 API 现在只暴露页面/方案/站点设置的读权限,不会泄漏询盘内容。

### Q10:首页/方案的 SEO 真的有效吗?

- **title / description** 100% 有效(搜索引擎直接读)。
- **keywords** 影响很小(Google 早不看,百度还象征性参考)。
- **ogImage** 决定社交分享卡片的颜值,**强烈建议每个页面配**。

| 用途 | URL |
|---|---|
| 后台首页 | `http://localhost:1337/admin` |
| 后台登录 | `http://localhost:1337/admin/auth/login` |
| 修改首页 | `http://localhost:1337/admin/content-manager/single-types/api::home-page.home-page` |
| 解决方案列表 | `http://localhost:1337/admin/content-manager/collection-types/api::solution.solution` |
| 询盘列表 | `http://localhost:1337/admin/content-manager/collection-types/api::inquiry.inquiry` |
| 媒体库 | `http://localhost:1337/admin/plugins/upload` |
| 管理员用户 | `http://localhost:1337/admin/settings/users` |
| 前端站点(开发) | `http://localhost:3000` |
| 公开 API 测试 | `http://localhost:1337/api/solutions` |
